# Sentry Integration

dotdo provides two ways to work with Sentry-compatible error tracking:

| | @dotdo/sentry | sentry.do |
|---|---------------|-----------|
| **Type** | OSS npm package | Managed edge service |
| **Transport** | Direct to Sentry APIs | Edge-optimized proxy |
| **Testing Mode** | InMemoryTransport | Full offline capture |
| **Use case** | Testing, development | Production workloads |
| **Infrastructure** | Zero | Zero (we manage it) |
| **Install** | `npm install @dotdo/sentry` | Deploy to Cloudflare |

## Quick Comparison

```typescript
// @dotdo/sentry - Drop-in replacement, great for tests
import * as Sentry from '@dotdo/sentry'
import { InMemoryTransport } from '@dotdo/sentry'

const transport = new InMemoryTransport()
Sentry.init({
  dsn: 'https://key@sentry.example.com/1',
  transport: () => transport,
})

Sentry.captureException(new Error('Test error'))
const events = transport.getEvents() // Access captured events
```

```typescript
// sentry.do - Managed edge service for production
import * as Sentry from 'sentry.do'

Sentry.init({
  dsn: env.SENTRY_DSN,
  environment: 'production',
  release: 'my-app@1.0.0',
})

Sentry.captureException(new Error('Production error'))
// Events routed through edge proxy with batching and retry
```

## When to Use Each

### Use @dotdo/sentry when:

- **Unit testing** - Capture and assert on error events without external calls
- **Development** - Full Sentry API with local-only mode
- **Edge functions** - Lightweight package (~15KB) for Cloudflare Workers
- **CI/CD pipelines** - Tests that need Sentry semantics without network

<Callout type="info">
@dotdo/sentry is a complete Sentry SDK replacement with edge compatibility. Perfect for tests and development with InMemoryTransport, or production use by forwarding to real Sentry APIs.
</Callout>

### Use sentry.do when:

- **Production applications** - Optimized edge proxy with request batching
- **Global edge deployment** - Sentry events routed through nearest edge location
- **High-volume workloads** - Intelligent sampling and rate limiting at the edge
- **Multi-tenant applications** - Per-tenant DSN routing and isolation

<Callout type="info">
sentry.do is Sentry on the Edge - a managed proxy service that optimizes error delivery, handles retries, and provides additional edge-native features.
</Callout>

## Migration Path

Start with `@dotdo/sentry` for development, then upgrade to `sentry.do` for production:

```typescript
// Development (package)
import * as Sentry from '@dotdo/sentry'

Sentry.init({
  dsn: env.SENTRY_DSN,
  environment: 'development',
})

// Production (service) - same API!
import * as Sentry from 'sentry.do'

Sentry.init({
  dsn: env.SENTRY_DSN,
  environment: 'production',
  release: 'my-app@1.0.0',
})

// Your application code stays the same
try {
  riskyOperation()
} catch (error) {
  Sentry.captureException(error)
}
```

Both packages share the same Sentry-compatible API, making migration seamless.
