# Intercom Integration

dotdo provides two ways to work with Intercom-compatible customer messaging APIs:

| | @dotdo/intercom | intercom.do |
|---|-----------------|-------------|
| **Type** | OSS npm package | Managed edge service |
| **Storage** | In-memory / local | Durable Object SQLite |
| **Persistence** | Configurable | Full durability |
| **Use case** | Testing, prototyping | Production workloads |
| **Infrastructure** | Zero | Zero (we manage it) |
| **Install** | `npm install @dotdo/intercom` | Deploy to Cloudflare |

## Quick Comparison

```typescript
// @dotdo/intercom - Local mode, great for tests
import { IntercomLocal } from '@dotdo/intercom'

const client = new IntercomLocal({
  workspaceId: 'test-workspace',
})
const contact = await client.contacts.create({
  role: 'user',
  email: 'alice@example.com',
})
// Data configurable - in-memory or local storage
```

```typescript
// intercom.do - Persistent edge service
import { IntercomClient } from 'intercom.do'

const client = new IntercomClient('https://your-worker.workers.dev')
const contact = await client.contacts.create({
  role: 'user',
  email: 'alice@example.com',
})
// Data persists globally at the edge
```

## When to Use Each

### Use @dotdo/intercom when:

- **Unit testing** - Fast, isolated tests without external dependencies
- **Prototyping** - Quick iteration without infrastructure setup
- **Intercom API proxy** - Connect to real Intercom API with familiar SDK
- **CI/CD pipelines** - Tests that need Intercom semantics
- **Local development** - Build customer messaging features offline

<Callout type="info">
@dotdo/intercom provides both an Intercom API client and a local implementation. Perfect for tests, development, and scenarios where you want Intercom-compatible APIs without external dependencies.
</Callout>

### Use intercom.do when:

- **Production applications** - Real customer data that must persist
- **Global edge deployment** - Messaging infrastructure close to users worldwide
- **AI/Agent workloads** - AI-powered support agents, automated responses
- **Real-time features** - Live chat, conversation streams, presence
- **Multi-channel support** - Email, in-app, and push notifications

<Callout type="info">
intercom.do is Customer Messaging on the Edge - a fully managed Intercom-compatible service running on Cloudflare's global network with zero infrastructure to manage.
</Callout>

## Migration Path

Start with `@dotdo/intercom` for development, then upgrade to `intercom.do` for production:

```typescript
// Development (package)
import { IntercomLocal } from '@dotdo/intercom'
const client = new IntercomLocal({ workspaceId: 'dev' })

// Production (service) - same API!
import { IntercomClient } from 'intercom.do'
const client = new IntercomClient('https://your-intercom.workers.dev')

// Your application code stays the same
const contact = await client.contacts.create({
  role: 'user',
  email: 'customer@example.com',
  name: 'Jane Smith',
})

const conversation = await client.conversations.create({
  from: { type: 'user', id: contact.id },
  body: 'Hi, I need help with my order',
})
```

Both packages share the same Intercom-compatible API, making migration seamless.
