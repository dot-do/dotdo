# Salesforce Integration

dotdo provides two ways to work with Salesforce CRM APIs:

| | @dotdo/salesforce | salesforce.do |
|---|-------------------|---------------|
| **Type** | OSS npm package | Managed edge service |
| **Backend** | Real Salesforce APIs | Edge-cached Salesforce proxy |
| **Runtime** | Edge-compatible | Cloudflare Workers |
| **Use case** | Drop-in jsforce replacement | Production CRM workflows |
| **Infrastructure** | Your Salesforce org | Zero (we manage it) |
| **Install** | `npm install @dotdo/salesforce` | Deploy to Cloudflare |

## Quick Comparison

```typescript
// @dotdo/salesforce - jsforce-compatible, connects to real Salesforce
import jsforce from '@dotdo/salesforce'

const conn = new jsforce.Connection({
  loginUrl: 'https://login.salesforce.com',
})

await conn.login('user@example.com', 'password+securitytoken')

const accounts = await conn.query(
  "SELECT Id, Name FROM Account WHERE Industry = 'Technology'"
)
```

```typescript
// salesforce.do - Edge-optimized CRM access
import { SalesforceClient } from 'salesforce.do'

const client = new SalesforceClient('https://your-worker.workers.dev')

const accounts = await client.query(
  "SELECT Id, Name FROM Account WHERE Industry = 'Technology'"
)
```

## When to Use Each

### Use @dotdo/salesforce when:

- **Migrating from jsforce** - Drop-in replacement with same API
- **Edge environments** - jsforce doesn't work in Cloudflare Workers
- **Unit testing** - Mock Salesforce responses without hitting the API
- **Gradual migration** - Keep existing jsforce code patterns

<Callout type="info">
@dotdo/salesforce is a jsforce-compatible library that works in edge environments. Use it when you need the familiar jsforce API but want edge compatibility.
</Callout>

### Use salesforce.do when:

- **Production CRM workflows** - Built for high-volume Salesforce operations
- **Edge-first architecture** - Native Cloudflare Workers integration
- **Caching and optimization** - Built-in response caching and request batching
- **AI agent integration** - MCP protocol support for LLM tool calling

<Callout type="info">
salesforce.do is Salesforce on the Edge - a managed service optimized for CRM workflows running on Cloudflare's global network.
</Callout>

## Migration Path

Start with `@dotdo/salesforce` as a drop-in jsforce replacement, then upgrade to `salesforce.do` for production:

```typescript
// Phase 1: Use @dotdo/salesforce (drop-in for jsforce)
import jsforce from '@dotdo/salesforce'

const conn = new jsforce.Connection({
  loginUrl: 'https://login.salesforce.com',
})

await conn.login(env.SF_USERNAME, env.SF_PASSWORD)

const accounts = await conn.query('SELECT Id, Name FROM Account')
```

```typescript
// Phase 2: Migrate to salesforce.do for zero-infrastructure
import { SalesforceClient } from 'salesforce.do'

const client = new SalesforceClient(env)

// Same query, with built-in caching and optimization
const accounts = await client.query('SELECT Id, Name FROM Account')
```
