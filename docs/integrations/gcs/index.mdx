# GCS Integration

dotdo provides two ways to work with GCS-compatible storage APIs:

| | @dotdo/gcs | gcs.do |
|---|------------|--------|
| **Type** | OSS npm package | Managed edge service |
| **Backend** | R2 or real GCS | Cloudflare R2 |
| **Persistence** | Full (R2-backed) | Full durability |
| **Use case** | Drop-in replacement, testing | Production edge storage |
| **Infrastructure** | Your R2 bucket | Zero (we manage it) |
| **Install** | `npm install @dotdo/gcs` | Deploy to Cloudflare |

## Quick Comparison

```typescript
// @dotdo/gcs - GCS SDK-compatible, backed by R2
import { Storage } from '@dotdo/gcs'

const storage = new Storage({ projectId: 'my-project' })
const bucket = storage.bucket('my-bucket')
const file = bucket.file('my-file.txt')
await file.save('Hello, World!')
```

```typescript
// gcs.do - Managed edge storage with GCS API
import { Storage } from 'gcs.do'

const storage = new Storage('https://your-worker.workers.dev')
const bucket = storage.bucket('my-bucket')
await bucket.file('my-file.txt').save('Hello, World!')
```

## When to Use Each

### Use @dotdo/gcs when:

- **Migrating from GCS** - Drop-in replacement for `@google-cloud/storage`
- **Testing** - Swap imports without changing code
- **Self-hosted R2** - Direct control over your R2 bucket bindings
- **Hybrid cloud** - Can also connect to real GCS when needed

<Callout type="info">
@dotdo/gcs is a GCS SDK-compatible package backed by Cloudflare R2. Perfect for migration from Google Cloud Storage - just swap the import.
</Callout>

### Use gcs.do when:

- **Zero infrastructure** - No R2 buckets to configure
- **Multi-tenant storage** - Built-in tenant isolation
- **Global edge delivery** - Automatic edge caching
- **Unified platform** - Integrated with mongo.do, kafka.do, etc.

<Callout type="info">
gcs.do is managed GCS-compatible storage on the edge. Zero egress fees, global distribution, and no infrastructure to manage.
</Callout>

## Migration Path

Start with `@dotdo/gcs` to migrate from Google Cloud Storage, then optionally upgrade to `gcs.do`:

```typescript
// Phase 1: Replace @google-cloud/storage with @dotdo/gcs
// Before
import { Storage } from '@google-cloud/storage'

// After - just change the import
import { Storage } from '@dotdo/gcs'

// All your existing code works unchanged
const storage = new Storage({ projectId: 'my-project' })
const bucket = storage.bucket('my-bucket')
const file = bucket.file('document.pdf')
await file.save(pdfBuffer, { contentType: 'application/pdf' })
```

```typescript
// Phase 2: Optionally migrate to gcs.do for managed service
import { Storage } from 'gcs.do'

const storage = new Storage('https://your-worker.workers.dev')
const bucket = storage.bucket('my-bucket')
// Same API, zero infrastructure
await bucket.file('document.pdf').save(pdfBuffer, {
  contentType: 'application/pdf',
})
```

## Why R2 Over GCS?

Both options use Cloudflare R2 as the storage backend, eliminating egress fees:

| Feature | Google Cloud Storage | Cloudflare R2 |
|---------|---------------------|---------------|
| Egress fees | $0.12/GB | **Free** |
| API compatibility | Native | S3/GCS-compatible |
| Edge access | Via CDN | Native edge access |
| Cold storage | Archive tiers | Automatic tiering |
| Min storage duration | 30-365 days | None |

Both `@dotdo/gcs` and `gcs.do` give you GCS API compatibility with R2 economics.
