# Stripe Integration

dotdo provides two ways to work with Stripe-compatible payment APIs:

| | @dotdo/stripe | stripe.do |
|---|---------------|-----------|
| **Type** | OSS npm package | Managed edge service |
| **Backend** | Real Stripe API or in-memory (StripeLocal) | Edge-native payment orchestration |
| **Persistence** | Via Stripe or in-memory (testing) | Durable Object storage + Stripe |
| **Use case** | Drop-in replacement, testing | Production edge workloads |
| **Infrastructure** | Zero | Zero (we manage it) |
| **Install** | `npm install @dotdo/stripe` | Deploy to Cloudflare |

## Quick Comparison

```typescript
// @dotdo/stripe - Drop-in Stripe SDK replacement
import { Stripe } from '@dotdo/stripe'

const stripe = new Stripe(env.STRIPE_SECRET_KEY)
const customer = await stripe.customers.create({
  email: 'customer@example.com',
})
// Works on edge, same API as official SDK
```

```typescript
// stripe.do - Managed edge payment service
import { StripeClient } from 'stripe.do'

const client = new StripeClient('https://your-worker.workers.dev')
const customer = await client.customers.create({
  email: 'customer@example.com',
})
// Zero-latency edge payments with built-in orchestration
```

## When to Use Each

### Use @dotdo/stripe when:

- **Migrating from Stripe SDK** - Drop-in replacement with zero code changes
- **Edge deployment** - Need Stripe in Cloudflare Workers (no Node.js)
- **Unit testing** - Use StripeLocal for fast, isolated tests without real API calls
- **CI/CD pipelines** - Tests that need Stripe semantics without network

<Callout type="info">
@dotdo/stripe is a Stripe SDK drop-in replacement that works on the edge. Use StripeLocal for in-memory testing, or connect to real Stripe APIs in production.
</Callout>

### Use stripe.do when:

- **Production applications** - Mission-critical payment processing
- **Global edge deployment** - Payment processing close to users worldwide
- **Advanced orchestration** - Multi-provider routing, retry policies, idempotency
- **AI/Agent workloads** - Automated billing, subscription management

<Callout type="info">
stripe.do is Stripe on the Edge - a fully managed payment service running on Cloudflare's global network with built-in orchestration and failover.
</Callout>

## Migration Path

Start with `@dotdo/stripe` for development and testing, then upgrade to `stripe.do` for production with advanced features:

```typescript
// Development (package)
import { Stripe } from '@dotdo/stripe'
const stripe = new Stripe(env.STRIPE_SECRET_KEY)

// Production (service) - same API!
import { StripeClient } from 'stripe.do'
const stripe = new StripeClient('https://your-stripe.workers.dev')

// Your application code stays the same
const customer = await stripe.customers.create({
  email: 'customer@example.com',
})
const subscription = await stripe.subscriptions.create({
  customer: customer.id,
  items: [{ price: 'price_xxx' }],
})
```

Both packages share a Stripe-compatible API, making migration seamless.
