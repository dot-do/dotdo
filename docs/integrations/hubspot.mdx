# HubSpot

Drop-in replacement for the HubSpot JavaScript SDK. Your existing `@hubspot/api-client` code works unchanged - just swap the import.

```typescript
// Before: HubSpot
import { Client } from '@hubspot/api-client'

// After: dotdo
import { Client } from '@dotdo/hubspot'

// Code stays the same
const hubspot = new Client({ accessToken: env.HUBSPOT_ACCESS_TOKEN })

const contact = await hubspot.crm.contacts.basicApi.create({
  properties: {
    email: 'customer@example.com',
    firstname: 'John',
    lastname: 'Doe',
  },
})

const deal = await hubspot.crm.deals.basicApi.create({
  properties: {
    dealname: 'Enterprise Contract',
    amount: '50000',
    dealstage: 'appointmentscheduled',
    pipeline: 'default',
  },
})
```

## Why hubspot.do?

| HubSpot SDK | @dotdo/hubspot |
|-------------|----------------|
| Node.js runtime required | Edge-compatible (Cloudflare Workers) |
| Real API calls in tests | Local testing with HubSpotCRM |
| Network latency | Zero latency in local mode |
| Rate limits on API | Unlimited requests locally |
| Requires account for dev | Works offline |

**This is a proxy/compatibility layer.** It can either forward to real HubSpot APIs (production) or run entirely locally for testing and development. The API surface matches HubSpot's official SDK.

## Features

### Implemented (CRM APIs)

**Contacts**
- `basicApi.create()`, `basicApi.getById()`, `basicApi.update()`, `basicApi.archive()`, `basicApi.getPage()`
- `searchApi.doSearch()` - Full-text and filter-based search
- `batchApi.create()`, `batchApi.read()`, `batchApi.update()`, `batchApi.archive()`

**Companies**
- Full CRUD operations via `basicApi`
- Search with filters via `searchApi`
- Batch operations via `batchApi`

**Deals**
- Full CRUD operations with pipeline stages
- Deal stage management
- Search and batch operations

**Tickets**
- Full CRUD operations
- Pipeline and stage support
- Search and batch operations

**Line Items & Products**
- Product catalog management
- Line items on deals
- Full CRUD operations

**Quotes**
- Quote creation and management
- Status tracking
- Associations with deals and contacts

**Associations (v4 API)**
- `associations.v4.basicApi.create()`, `getPage()`, `archive()`
- `associations.v4.batchApi.create()`, `read()`, `archive()`
- Contact-to-company, deal-to-contact, and custom associations

**Engagements**
- Notes, Emails, Calls, Meetings
- Full CRUD via `crm.objects`

**Properties API**
- `properties.coreApi.getAll()`, `getByName()`, `create()`, `update()`, `archive()`
- `properties.groupsApi` for property groups
- Custom property definitions

**Schemas API (Custom Objects)**
- `schemas.coreApi.getAll()`, `getById()`, `create()`, `update()`, `archive()`
- Define custom CRM objects

**Owners API**
- `crm.owners.getPage()`, `getById()`
- HubSpot user management

**Pipelines API**
- `pipelines.pipelinesApi` for pipeline management
- `pipelines.pipelineStagesApi` for stages

**Workflows**
- Workflow creation and management
- Enrollment triggers (form submission, property change, list membership)
- Actions (set property, send email, delay, branch, webhook)
- Contact enrollment and unenrollment
- Workflow analytics and stats

**Forms API**
- `HubSpotForms.createForm()`, `getForm()`, `updateForm()`, `deleteForm()`
- Form publishing and state management
- Form submissions with validation
- Field groups and field management
- Form statistics and analytics

**Webhooks API**
- `HubSpotWebhooks.subscribe()`, `unsubscribe()`, `updateSubscription()`
- Event triggering and delivery
- Signature verification (v1 and v2)
- Delivery retry with exponential backoff
- Event history and delivery logs

### Not Yet Implemented

- Marketing Emails API
- Landing Pages API
- Blog API
- Files API
- OAuth helpers

## Quick Start

### Install

```bash
npm install @dotdo/hubspot
```

### Basic Usage

```typescript
import { Client } from '@dotdo/hubspot'

const hubspot = new Client({ accessToken: env.HUBSPOT_ACCESS_TOKEN })

// Create a contact
const contact = await hubspot.crm.contacts.basicApi.create({
  properties: {
    email: 'lead@example.com',
    firstname: 'Jane',
    lastname: 'Smith',
    phone: '+1-555-0123',
    company: 'Acme Corp',
    lifecyclestage: 'lead',
  },
})

// Create a company
const company = await hubspot.crm.companies.basicApi.create({
  properties: {
    name: 'Acme Corp',
    domain: 'acme.com',
    industry: 'Technology',
    numberofemployees: '500',
  },
})

// Associate contact to company
await hubspot.crm.associations.v4.basicApi.create(
  'contacts',
  contact.id,
  'companies',
  company.id,
  [{ associationCategory: 'HUBSPOT_DEFINED', associationTypeId: 1 }]
)

// Search for contacts
const results = await hubspot.crm.contacts.searchApi.doSearch({
  filterGroups: [{
    filters: [{
      propertyName: 'lifecyclestage',
      operator: 'EQ',
      value: 'lead',
    }],
  }],
  sorts: [{ propertyName: 'createdate', direction: 'DESCENDING' }],
  limit: 50,
})

console.log(`Found ${results.total} leads`)
```

```typescript
  name: 'Test Corp',
```

### Edge Compatibility

Works in Cloudflare Workers without Node.js dependencies:

```typescript
// worker.ts
import { Client } from '@dotdo/hubspot'

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const hubspot = new Client({ accessToken: env.HUBSPOT_ACCESS_TOKEN })

    const url = new URL(request.url)

    if (url.pathname === '/api/contacts' && request.method === 'POST') {
      const body = await request.json()

      const contact = await hubspot.crm.contacts.basicApi.create({
        properties: {
          email: body.email,
          firstname: body.firstName,
          lastname: body.lastName,
        },
      })

      return Response.json(contact)
    }

    return new Response('Not Found', { status: 404 })
  },
}
```

## Search Operators

| Operator | Description |
|----------|-------------|
| `EQ` | Equal to |
| `NEQ` | Not equal to |
| `LT` | Less than |
| `LTE` | Less than or equal |
| `GT` | Greater than |
| `GTE` | Greater than or equal |
| `BETWEEN` | Between two values |
| `IN` | In list of values |
| `NOT_IN` | Not in list |
| `HAS_PROPERTY` | Property exists |
| `NOT_HAS_PROPERTY` | Property doesn't exist |
| `CONTAINS_TOKEN` | Contains text |
| `NOT_CONTAINS_TOKEN` | Doesn't contain text |

```typescript
    })),
  })

  console.log(`Created ${result.results.length} contacts`)
  if (result.errors.length > 0) {
    console.error(`Failed: ${result.errors.length}`)
  }
}
```
