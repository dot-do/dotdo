# Installation

Two minutes. That's all you need.

## Create a New Project

The fastest way to get started is with `npx dotdo init`:

```bash
npx dotdo init my-startup
cd my-startup
npm install
```

This creates a complete project structure:

```
my-startup/
  src/
    index.ts        # Your DO class and worker entry point
  wrangler.jsonc    # Cloudflare configuration
  package.json
  tsconfig.json
```

<Callout type="info">
**Already have a project?** You can add dotdo to an existing project:

<Tabs items={['npm', 'pnpm', 'yarn', 'bun']}>
  <Tab value="npm">
```bash
npm install dotdo
```
  </Tab>

  <Tab value="bun">

  </Tab>
</Tabs>
</Callout>

## Project Configuration

The init command creates `wrangler.jsonc` with sensible defaults:

```json title="wrangler.jsonc"
{
  "name": "my-startup",
  "main": "src/index.ts",
  "compatibility_date": "2024-12-01",
  "compatibility_flags": ["nodejs_compat"],
  "durable_objects": {
    "bindings": [
      { "name": "DO", "class_name": "MyStartup" }
    ]
  },
  "migrations": [
    { "tag": "v1", "new_sqlite_classes": ["MyStartup"] }
  ]
}
```

<Callout type="warn">
**First time using Wrangler?** Run `npx wrangler login` to authenticate with your Cloudflare account.
</Callout>

## Verify Installation

```bash
npm run dev
```

You should see output similar to:

```
Starting local server...
Ready on http://localhost:8787
```

Test your API by making a request:

```bash
curl http://localhost:8787
```

You should see a response from your Durable Object.

## Environment Variables

Create a `.dev.vars` file for local secrets:

```bash title=".dev.vars"
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
```

<Callout type="warn">
**These keys are required.** Without at least one API key, the named agents (Priya, Ralph, Tom, etc.) can't call LLMs and your startup won't work. See [API Keys](/docs/getting-started/api-keys) for details on getting and configuring these keys.
</Callout>

<Callout type="error">
Never commit `.dev.vars`. It's already in `.gitignore`, but double-check.
</Callout>

For production, use Wrangler secrets:

```bash
npx wrangler secret put OPENAI_API_KEY
npx wrangler secret put ANTHROPIC_API_KEY
```

## TypeScript Configuration

The init command creates the right `tsconfig.json`, but here's what matters:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "types": ["@cloudflare/workers-types"]
  }
}
```

Key settings:

| Option | Value | Why |
|--------|-------|-----|
| `target` | ES2022 | V8 isolates support modern JS |
| `moduleResolution` | bundler | Wrangler bundles your code |
| `types` | workers-types | Cloudflare globals like `DurableObject` |

## Project Structure

After init, here's what you're working with:

```
my-startup/
  src/
    index.ts        # Your DO class and worker entry point
  wrangler.jsonc    # Cloudflare deployment config
  .dev.vars         # Local secrets (gitignored)
  package.json
  tsconfig.json
```

The key file is `src/index.ts`:

```typescript title="src/index.ts"
import { DO } from 'dotdo'

export class MyStartup extends DO {
  static readonly $type = 'MyStartup'
}

// Re-export the simple worker as default
export { default } from 'dotdo/workers/simple'
```

This gives you a basic Durable Object. To build a full AI-powered startup, extend it with agents:

```typescript title="src/index.ts"
import { Startup } from 'dotdo'
import { priya, ralph, tom, mark, sally } from 'agents.do'

export class MyStartup extends Startup {
  hypothesis = 'Your hypothesis here'

  async launch() {
    const spec = priya`define the MVP for ${this.hypothesis}`
    let app = ralph`build ${spec}`

    do {
      app = ralph`improve ${app} per ${tom}`
    } while (!await tom.approve(app))

    mark`announce the launch`
    sally`start selling`
  }
}
```

## Import Aliases

The dotdo ecosystem uses domain-style imports like `agents.do` and `humans.do`. These are **path aliases** that need to be configured in your project.

<Callout type="warn">
**Required for named agents and human escalation.** Without these aliases, you'll get "Cannot find module 'agents.do'" errors when using code from the quickstart guide.
</Callout>

### Automatic Setup

If you used `npx dotdo init`, these are already configured. Skip to the next section.

### Manual Setup

Add path mappings to your `tsconfig.json`:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "paths": {
      "agents.do": ["./node_modules/dotdo/dist/agents/named/index.js"],
      "humans.do": ["./node_modules/dotdo/dist/lib/humans/index.js"]
    }
  }
}
```

Or use Node.js subpath imports in `package.json`:

```json title="package.json"
{
  "imports": {
    "agents.do": "dotdo/agents/named",
    "humans.do": "dotdo/lib/humans"
  }
}
```

Now you can use the domain-style imports:

```typescript
import { priya, ralph, tom, mark, sally, quinn } from 'agents.do'
import { ceo, legal, cfo, cto, hr, support } from 'humans.do'
```

## Troubleshooting

### "Cannot find module 'agents.do'" or "Cannot find module 'humans.do'"

These are path aliases, not npm packages. Add the path mappings to your `tsconfig.json` or `package.json` as shown in the [Import Aliases](#import-aliases) section above.

### "Cannot find module 'dotdo'"

Make sure you're in the project directory and ran `npm install`.

### "Wrangler not authenticated"

Run `npx wrangler login` and follow the browser prompts.

### "Durable Object not found"

Check that your `wrangler.jsonc` has the correct class name in the `durable_objects` binding.

### Port 8787 already in use

Another Wrangler instance is running. Kill it or use a different port:

```bash
npm run dev -- --port 8788
```
