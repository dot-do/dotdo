# linear.do

**Linear on the Edge** — A Linear-compatible project management service that runs entirely on Cloudflare Workers, with native AI agent support, real-time webhooks, and global edge deployment.

```typescript
import { LinearClient } from 'linear.do'

const client = new LinearClient('https://your-worker.workers.dev')

// It's just Linear
const { team } = await client.teamCreate({ name: 'Engineering', key: 'ENG' })
const { issue } = await client.issueCreate({
  teamId: team.id,
  title: 'Implement authentication',
  priority: 1,
})
```

<Callout type="info">
Looking for an in-memory implementation for testing? See [@dotdo/linear](/docs/integrations/linear/package) for a zero-dependency Linear SDK.
</Callout>

## Why linear.do?

Traditional project management tools require external API calls, connection management, and careful rate limiting. linear.do eliminates all of that by running directly on Cloudflare's edge network:

- **Zero Infrastructure** — No servers to manage, no connection limits, no cold starts
- **Global by Default** — Data lives at the edge, close to your users
- **Linear Compatible** — Drop-in replacement for Linear API operations
- **AI-Native** — Built-in support for AI agents, MCP protocol, and automated workflows
- **Serverless Economics** — Pay only for what you use, scale to zero

const linear = new LinearClient('https://your-worker.workers.dev')

const response = await anthropic.messages.create({

### Automated Sprint Planning

```typescript
import { LinearClient } from 'linear.do'

const linear = new LinearClient('https://your-worker.workers.dev')

async function planSprint(teamId: string) {
  // Get unassigned issues ordered by priority
  const backlog = await linear.issues({
    filter: {
      team: { id: { eq: teamId } },
      assignee: { null: true },
      state: { type: { eq: 'backlog' } },
    },
    orderBy: 'priority',
  })

  // Get team members and their current workload
  const team = await linear.team(teamId)
  const members = await team.members?.()

  const assignments = await Promise.all(
    members?.nodes.map(async (member) => {
      const inProgress = await linear.issues({
        filter: {
          assignee: { id: { eq: member.id } },
          state: { type: { eq: 'started' } },
        },
      })
      return { member, workload: inProgress.nodes.length }
    }) ?? []
  )

  // Auto-assign based on capacity
  for (const issue of backlog.nodes.slice(0, 10)) {
    const leastBusy = assignments.sort((a, b) => a.workload - b.workload)[0]
    await linear.issueUpdate(issue.id, {
      assigneeId: leastBusy.member.id,
    })
    leastBusy.workload++
  }
}
```

### Real-Time Webhooks

```typescript
import { LinearClient } from 'linear.do'

const linear = new LinearClient('https://your-worker.workers.dev')

// Register webhook
await linear.webhookCreate({
  url: 'https://your-app.com/webhooks/linear',
  resourceTypes: ['Issue', 'Comment', 'Project'],
  secret: env.WEBHOOK_SECRET,
})

// Handle incoming webhooks
export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const payload = await request.text()

    const isValid = await linear.webhooks.verifyRequest(
      payload,
      {
        'linear-signature': request.headers.get('linear-signature')!,
        'linear-timestamp': request.headers.get('linear-timestamp')!,
      },
      env.WEBHOOK_SECRET
    )

    if (!isValid) {
      return new Response('Invalid signature', { status: 401 })
    }

    const event = JSON.parse(payload)

    switch (event.type) {
      case 'Issue':
        await handleIssueEvent(event)
        break
      case 'Comment':
        await handleCommentEvent(event)
        break
    }

    return new Response('OK')
  }
}
```

### Sprint Metrics Dashboard

```typescript
async function getSprintMetrics(cycleId: string) {
  const cycle = await linear.cycle(cycleId)
  const issues = await cycle.issues?.()

  const total = issues?.nodes.length ?? 0
  const completed = issues?.nodes.filter(
    (i) => i.state.type === 'completed'
  ).length ?? 0
  const inProgress = issues?.nodes.filter(
    (i) => i.state.type === 'started'
  ).length ?? 0

  return {
    total,
    completed,
    inProgress,
    remaining: total - completed,
    progress: total > 0 ? (completed / total) * 100 : 0,
    daysLeft: Math.ceil(
      (new Date(cycle.endsAt).getTime() - Date.now()) / (24 * 60 * 60 * 1000)
    ),
    burndownRate: completed / (total - (total - completed - inProgress) || 1),
  }
}
```

## Architecture

```
+---------------------------------------------------------------------------+
|                           Client Applications                              |
+-------------------+-------------------+-------------------+----------------+
|    GraphQL API    |     HTTP/RPC      |    Webhooks       | Service Bind   |
|  (Linear compat)  |    JSON-RPC       |   Real-time       | Worker-to-Wkr  |
+-------------------+-------------------+-------------------+----------------+
|                          linear.do Worker (Edge)                          |
+---------------------------------------------------------------------------+
|  Query Engine  |  Webhook Dispatcher  |  MCP Server  |  Time-Travel       |
+---------------------------------------------------------------------------+
|                    Durable Objects (SQLite Storage)                       |
+-----------------------------------+---------------------------------------+
|           TemporalStore           |              WindowManager            |
|    (Issues, Projects, Teams)      |     (Cycles, Sprint Tracking)         |
+-----------------------------------+---------------------------------------+
```

linear.do stores all data in Durable Objects with SQLite:

1. **TemporalStore** — Version-controlled storage for issues, projects, and teams
2. **WindowManager** — Time-windowed tracking for cycles and sprints
3. **Edge Execution** — Queries execute at the edge, close to your users
4. **Webhook System** — Real-time event dispatch with retry and verification

## Common Patterns

### Issue Triage Workflow

```typescript
async function triageIssue(issueId: string) {
  const issue = await linear.issue(issueId)

  // Auto-assign priority based on keywords
  let priority = 3 // Medium default

  if (issue.title.toLowerCase().includes('crash') ||
      issue.title.toLowerCase().includes('down')) {
    priority = 1 // Urgent
  } else if (issue.title.toLowerCase().includes('bug')) {
    priority = 2 // High
  }

  // Auto-assign labels
  const labels: string[] = []
  if (issue.description?.includes('mobile')) labels.push('mobile')
  if (issue.description?.includes('api')) labels.push('backend')

  await linear.issueUpdate(issueId, {
    priority,
    labelIds: labels,
  })
}
```

### Cycle Progress Tracking

```typescript
async function trackCycleProgress(teamId: string) {
  const team = await linear.team(teamId)
  const cycle = await team.activeCycle

  if (!cycle) return null

  const issues = await cycle.issues?.()
  const states = issues?.nodes.reduce((acc, issue) => {
    acc[issue.state.type] = (acc[issue.state.type] || 0) + 1
    return acc
  }, {} as Record<string, number>)

  return {
    cycleId: cycle.id,
    name: cycle.name,
    startsAt: cycle.startsAt,
    endsAt: cycle.endsAt,
    issuesByState: states,
    completionRate: (states?.completed || 0) / (issues?.nodes.length || 1),
  }
}
```
