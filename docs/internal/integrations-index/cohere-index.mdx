# Cohere Integration

dotdo provides two ways to work with Cohere's AI APIs:

| | @dotdo/cohere | cohere.do |
|---|---------------|-----------|
| **Type** | OSS npm package | Managed edge service |
| **Runtime** | Any JavaScript runtime | Cloudflare Workers |
| **API Backend** | Cohere API (your key) | Cohere API (managed) |
| **Use case** | Testing, development | Production workloads |
| **Key management** | Your responsibility | Handled by service |
| **Install** | `npm install @dotdo/cohere` | Deploy to Cloudflare |

## Quick Comparison

```typescript
// @dotdo/cohere - Your API key, edge-compatible SDK
import { CohereClient } from '@dotdo/cohere'

const cohere = new CohereClient({ token: process.env.COHERE_API_KEY })
const response = await cohere.generate({
  prompt: 'Write a tagline for an ice cream shop',
  model: 'command',
})
```

```typescript
// cohere.do - Managed service with built-in key management
import { CohereClient } from 'cohere.do'

const cohere = new CohereClient('https://your-cohere.workers.dev')
const response = await cohere.generate({
  prompt: 'Write a tagline for an ice cream shop',
  model: 'command',
})
// API key securely stored in Workers secrets
```

## When to Use Each

### Use @dotdo/cohere when:

- **Unit testing** - Mock responses without external dependencies
- **Local development** - Quick iteration with your own API key
- **Edge compatibility** - Need Cohere SDK in Workers, Deno, or Bun
- **Custom proxies** - Route requests through your own infrastructure
- **Cost control** - Direct billing relationship with Cohere

<Callout type="info">
@dotdo/cohere is a drop-in replacement for the official `cohere-ai` SDK. Your existing code works unchanged - just swap the import.
</Callout>

### Use cohere.do when:

- **Production applications** - Managed infrastructure with monitoring
- **Team environments** - Centralized API key management
- **Multi-tenant SaaS** - Per-tenant usage tracking and billing
- **Global edge deployment** - Requests proxied from nearest edge location
- **Agent integration** - Native support for dotdo named agents

<Callout type="info">
cohere.do is Cohere on the Edge - a managed service that handles API key rotation, rate limiting, usage analytics, and integrates with dotdo's agent system.
</Callout>

## Migration Path

Start with `@dotdo/cohere` for development, then upgrade to `cohere.do` for production:

```typescript
// Development (package) - your API key
import { CohereClient } from '@dotdo/cohere'
const cohere = new CohereClient({ token: process.env.COHERE_API_KEY })

// Production (service) - managed keys
import { CohereClient } from 'cohere.do'
const cohere = new CohereClient('https://your-cohere.workers.dev')

// Your application code stays the same
const response = await cohere.generate({
  prompt: 'Explain quantum computing',
  model: 'command',
  maxTokens: 200,
})
```

Both packages share the same Cohere-compatible API, making migration seamless.

## Integration with Named Agents

Both packages work with dotdo's named agents for specialized AI personas:

```typescript
import { priya, ralph, tom, mark, sally, quinn } from 'agents.do'

// Named agents can use Cohere models under the hood
const spec = await priya`define the MVP for ${hypothesis}`
let app = await ralph`build ${spec}`

do {
  app = await ralph`improve ${app} per ${tom}`
} while (!await tom.approve(app))

await mark`announce the launch of ${app}`
await sally`start selling ${app}`
```
