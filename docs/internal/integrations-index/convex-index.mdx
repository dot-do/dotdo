# Convex Integration

dotdo provides two ways to work with Convex-compatible reactive backends:

| | @dotdo/convex | convex.do |
|---|---------------|-----------|
| **Type** | OSS npm package | Managed edge service |
| **Storage** | Durable Object SQLite | Durable Object SQLite |
| **Persistence** | Local development | Full global durability |
| **Use case** | Testing, development | Production workloads |
| **Infrastructure** | Zero | Zero (we manage it) |
| **Install** | `npm install @dotdo/convex` | Deploy to Cloudflare |

## Quick Comparison

```typescript
// @dotdo/convex - Drop-in replacement for Convex browser SDK
import { ConvexClient } from '@dotdo/convex'

const client = new ConvexClient('https://your-deployment.convex.cloud')

// Real-time subscription
const unsubscribe = client.onUpdate(api.messages.list, { channel: 'general' }, (messages) => {
  console.log('Messages:', messages)
})

// Mutation with optimistic update
await client.mutation(api.messages.send, { body: 'Hello!' })
```

```typescript
// convex.do - Managed edge-native reactive backend
import { ConvexClient } from 'convex.do'

const client = new ConvexClient('https://your-worker.workers.dev')

// Same API, global edge deployment
const unsubscribe = client.onUpdate(api.messages.list, { channel: 'general' }, (messages) => {
  console.log('Messages:', messages)
})

await client.mutation(api.messages.send, { body: 'Hello!' })
```

## When to Use Each

### Use @dotdo/convex when:

- **Unit testing** - Fast, isolated tests without external dependencies
- **Local development** - Develop without connecting to cloud services
- **Migrating from Convex** - Drop-in replacement with same API
- **Edge functions** - Need Convex API semantics in edge environments

<Callout type="info">
@dotdo/convex is a Convex browser SDK-compatible implementation. It can run entirely locally on Durable Object SQLite storage or connect to real Convex APIs in production.
</Callout>

### Use convex.do when:

- **Production applications** - Real user data with global persistence
- **Global edge deployment** - Data and compute close to users worldwide
- **Real-time features** - Subscriptions, optimistic updates at scale
- **Zero infrastructure** - No servers to manage, automatic scaling

<Callout type="info">
convex.do is a Convex-compatible reactive backend running on Cloudflare's global network with zero infrastructure to manage.
</Callout>

## Migration Path

Start with `@dotdo/convex` for development, then upgrade to `convex.do` for production:

```typescript
// Development (package) - local or proxied
import { ConvexClient } from '@dotdo/convex'

const client = new ConvexClient('https://your-deployment.convex.cloud', {
  doNamespace: env.CONVEX,
})

// Production (service) - same API!
import { ConvexClient } from 'convex.do'

const client = new ConvexClient('https://your-convex.workers.dev')

// Your application code stays the same
const products = await client.query(api.products.list, {})
await client.mutation(api.products.create, { name: 'Widget', price: 29.99 })
```

Both packages share the same Convex-compatible API, making migration seamless.
