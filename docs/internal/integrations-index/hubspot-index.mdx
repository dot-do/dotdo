# HubSpot Integration

dotdo provides two ways to work with HubSpot-compatible APIs:

| | @dotdo/hubspot | hubspot.do |
|---|----------------|------------|
| **Type** | OSS npm package | Managed edge service |
| **Storage** | In-memory (HubSpotCRM) | Durable Object SQLite |
| **Persistence** | None (resets on restart) | Full durability |
| **Use case** | Testing, prototyping | Production workloads |
| **Infrastructure** | Zero | Zero (we manage it) |
| **Install** | `npm install @dotdo/hubspot` | Deploy to Cloudflare |

## Quick Comparison

```typescript
// @dotdo/hubspot - Drop-in SDK replacement, great for tests
import { Client } from '@dotdo/hubspot'

const hubspot = new Client({ accessToken: env.HUBSPOT_ACCESS_TOKEN })
const contact = await hubspot.crm.contacts.basicApi.create({
  properties: { email: 'alice@example.com', firstname: 'Alice' },
})
// Forwards to real HubSpot API (or use HubSpotCRM for local mode)
```

```typescript
// hubspot.do - Persistent edge CRM
import { HubSpotClient } from 'hubspot.do'

const client = new HubSpotClient('https://your-worker.workers.dev')
const contact = await client.crm.contacts.create({
  email: 'alice@example.com',
  firstname: 'Alice',
})
// Data persists globally on the edge
```

## When to Use Each

### Use @dotdo/hubspot when:

- **Unit testing** - Fast, isolated tests without HubSpot API calls
- **Prototyping** - Quick iteration without rate limits
- **Edge functions** - Need HubSpot SDK API on Cloudflare Workers
- **CI/CD pipelines** - Tests that need HubSpot semantics
- **Migrating from HubSpot SDK** - Drop-in replacement for `@hubspot/api-client`

<Callout type="info">
@dotdo/hubspot provides both a production-ready HubSpot API client (edge-compatible) and an in-memory HubSpotCRM for local development. Perfect for tests and development without API rate limits.
</Callout>

### Use hubspot.do when:

- **Production applications** - Real CRM data that must persist
- **Global edge deployment** - Data close to users worldwide
- **AI/Agent workloads** - CRM operations with agent integration
- **Real-time features** - Webhooks, workflow automation
- **Multi-tenant SaaS** - Isolated CRM per tenant

<Callout type="info">
hubspot.do is HubSpot CRM on the Edge - a fully managed service running on Cloudflare's global network with zero infrastructure to manage.
</Callout>

## Migration Path

Start with `@dotdo/hubspot` for development, then upgrade to `hubspot.do` for production:

```typescript
// Development (package) - Using HubSpotCRM for local mode
import { HubSpotCRM } from '@dotdo/hubspot/crm'
const crm = new HubSpotCRM(ctx.storage)
const contact = await crm.createContact({
  email: 'test@example.com',
  firstName: 'John',
})

// Production (service) - Same operations, persistent storage
import { HubSpotClient } from 'hubspot.do'
const client = new HubSpotClient('https://your-hubspot.workers.dev')
const contact = await client.crm.contacts.create({
  email: 'test@example.com',
  firstName: 'John',
})
```

The package's local mode (`HubSpotCRM`) provides the same API as the managed service, making migration seamless.
