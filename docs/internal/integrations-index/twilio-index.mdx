# Twilio Integration

dotdo provides two ways to work with Twilio-compatible APIs:

| | @dotdo/twilio | twilio.do |
|---|---------------|-----------|
| **Type** | OSS npm package | Managed edge service |
| **Backend** | Real Twilio API or mock mode | Cloudflare Workers + Twilio |
| **Use case** | Testing, development, direct API calls | Production workloads, webhook handling |
| **Infrastructure** | Zero | Zero (we manage it) |
| **Install** | `npm install @dotdo/twilio` | Deploy to Cloudflare |

## Quick Comparison

```typescript
// @dotdo/twilio - Drop-in replacement for Twilio SDK
import twilio from '@dotdo/twilio'

const client = twilio('ACXXXXXXXXX', 'your_auth_token')
const message = await client.messages.create({
  body: 'Hello from dotdo!',
  from: '+15017122661',
  to: '+15558675310',
})
// Works in tests (mock mode) or production (real Twilio)
```

```typescript
// twilio.do - Managed edge service
import { TwilioClient } from 'twilio.do'

const client = new TwilioClient('https://your-worker.workers.dev')
await client.sms.send({
  to: '+15558675310',
  body: 'Hello from the edge!',
})
// Managed webhooks, built-in analytics, global edge deployment
```

## When to Use Each

### Use @dotdo/twilio when:

- **Unit testing** - Fast, isolated tests without API calls
- **Development** - Local testing with input validation
- **Edge functions** - Need Twilio SDK API but edge-compatible
- **CI/CD pipelines** - Tests that need Twilio semantics without credentials

<Callout type="info">
@dotdo/twilio is a drop-in replacement for the official Twilio SDK. It validates inputs locally and can operate in mock mode for testing, or connect to real Twilio APIs for production.
</Callout>

### Use twilio.do when:

- **Production applications** - Managed webhook handling and delivery
- **Global edge deployment** - Low-latency SMS/Voice from 300+ cities
- **AI/Agent workloads** - Built-in conversation state, MCP protocol
- **Multi-channel communications** - Unified SMS, Voice, WhatsApp, Verify

<Callout type="info">
twilio.do is a fully managed Twilio proxy running on Cloudflare's global network. Handles webhook signatures, rate limiting, and provides built-in analytics.
</Callout>

## Migration Path

Start with `@dotdo/twilio` for development, then optionally upgrade to `twilio.do` for production:

```typescript
// Development (package) - Direct Twilio API
import twilio from '@dotdo/twilio'
const client = twilio(env.TWILIO_ACCOUNT_SID, env.TWILIO_AUTH_TOKEN)
await client.messages.create({
  body: 'Hello!',
  from: '+15017122661',
  to: '+15558675310',
})

// Production (service) - Managed edge proxy
import { TwilioClient } from 'twilio.do'
const client = new TwilioClient('https://your-twilio.workers.dev')
await client.sms.send({
  to: '+15558675310',
  body: 'Hello!',
})
```

Both packages provide Twilio-compatible APIs, making it easy to start with the OSS package and upgrade to the managed service when you need production features like automatic webhook handling, analytics, and conversation state.
