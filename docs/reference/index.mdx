# API Reference

This reference section documents all public APIs in the dotdo runtime. Use this as a comprehensive guide when building with dotdo Durable Objects.

## How to Navigate

The API reference is organized by DO class hierarchy and core interfaces:

### DO Class Hierarchy

dotdo provides a hierarchy of Durable Object classes, each adding capabilities on top of the previous:

| Class | Size | Capabilities | Reference |
|-------|------|--------------|-----------|
| [DOCore](/reference/do-core) | ~5KB | State, alarms, routing, WebSockets | Base class |
| [DOSemantic](/reference/do-semantic) | ~20KB | Nouns, Verbs, Things, Actions | Extends DOCore |
| [DOStorage](/reference/do-storage) | ~40KB | InMemory + Pipeline + SQLite + Iceberg | Extends DOSemantic |
| [DOWorkflow](/reference/do-workflow) | ~60KB | WorkflowContext ($), cascade, scheduling | Extends DOStorage |
| [DOFull](/reference/do-full) | ~80KB | AI, human-in-loop, fanout, streaming | Full runtime |

Choose the smallest class that provides the capabilities you need to minimize cold start time and memory usage.

### Core Interfaces

| Interface | Description | Reference |
|-----------|-------------|-----------|
| [WorkflowContext ($)](/reference/workflow-context) | Unified DSL for events, scheduling, and durable execution | `createWorkflowContext()` |
| [Types](/reference/types) | TypeScript type definitions for all public APIs | `Thing`, `Noun`, `Verb`, etc. |

## Import Paths

All public APIs are exported from the `dotdo/core` entry point:

```typescript
// DO Classes
import { DOCore, DOSemantic, DOStorage, DOWorkflow, DOFull } from 'dotdo/core'

// Types
import type { Thing, Noun, Verb, ActionResult } from 'dotdo/core'
import type { WorkflowContext, Event, CascadeOptions } from 'dotdo/core'

// Environment types (for wrangler.toml bindings)
import type { DOCoreEnv, DOSemanticEnv, DOStorageEnv, DOWorkflowEnv, DOFullEnv } from 'dotdo/core'

// Factory functions
import { createWorkflowContext } from 'dotdo/core'
```

## TypeScript Support

dotdo is written in TypeScript and provides full type definitions. Enable strict mode for the best experience:

```json
{
  "compilerOptions": {
    "strict": true,
    "moduleResolution": "bundler",
    "types": ["@cloudflare/workers-types"]
  }
}
```

### RPC Type Safety

When using RPC to call DO methods, TypeScript provides full inference:

```typescript
const stub = env.DOSemantic.get(id)

// Types are inferred from the DO class
const noun = await stub.defineNoun('Customer')
// noun: Noun (automatically typed)

const thing = await stub.createThing('Customer', { name: 'Alice' })
// thing: Thing<{ name: string }>
```

## Version Compatibility

This reference documents dotdo v1.x. The API follows semantic versioning:

- **Patch releases** (1.0.x) - Bug fixes, no API changes
- **Minor releases** (1.x.0) - New features, backwards compatible
- **Major releases** (x.0.0) - Breaking changes (with migration guide)
