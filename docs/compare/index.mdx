# Platform Comparisons

dotdo is a unique combination: **Cloudflare's edge runtime** + **AI agents as first-class citizens** + **human escalation built-in**. This section compares dotdo to platforms you might already know.

## At a Glance

| Platform | Best For | dotdo Advantage |
|----------|----------|-----------------|
| [Vercel](/docs/compare/vs-vercel) | Frontend deployment | Built-in state, AI agents, human escalation |
| [Firebase](/docs/compare/vs-firebase) | Real-time BaaS | Edge performance, predictable pricing, agents |
| [Temporal](/docs/compare/vs-temporal) | Durable workflows | Simpler model, no replay, managed infrastructure |
| [Cloudflare Workers](/docs/compare/vs-cloudflare-workers) | Low-level edge compute | Higher-level abstractions, 40+ compat SDKs |

## Quick Decision Guide

### Choose dotdo If

- You're building **AI-powered products** where agents do real work
- You need **human escalation** built into your workflows
- You want **edge performance** with 0ms cold starts
- You prefer **TypeScript** and familiar APIs (Postgres, Redis, etc.)
- You're building **multi-tenant SaaS** with per-tenant isolation
- You want **predictable pricing** without surprise bills

### Consider Alternatives If

- **Vercel**: You're deploying a static site or Next.js frontend only
- **Firebase**: You're building a mobile app with Google ecosystem integration
- **Temporal**: You need exactly-once semantics with complex saga patterns
- **Raw Workers**: You want maximum control with no framework overhead

## Common Migration Paths

### From Vercel + External Services

```typescript
// Before: Vercel + Neon + Upstash + Clerk + SendGrid
import { neon } from '@neondatabase/serverless'
import { Redis } from '@upstash/redis'
// ... multiple services, multiple bills

// After: dotdo (everything built-in)
export class MyApp extends DO {
  // Database is local SQLite
  // Cache is in-memory + SQLite
  // Auth is org.ai
  // Email is built-in channel
}
```

### From Firebase

```typescript
// Before: Firebase Admin SDK
import { initializeApp } from 'firebase-admin/app'
import { getFirestore } from 'firebase-admin/firestore'

// After: dotdo Firebase compat
import { initializeApp, getFirestore } from '@dotdo/firebase'
// Same API, runs on edge
```

### From Temporal

```typescript
// Before: Temporal workflow
export async function myWorkflow() {
  // Determinism required, replay-based
  await proxyActivities().doSomething()
}

// After: dotdo workflow
export class MyWorkflow extends DO {
  // No determinism required, checkpoint-based
  async run() {
    await $.do('doSomething')
  }
}
```

### From Raw Workers

```typescript
// Before: Manual everything
export class MyDO extends DurableObject {
  // Manual schemas, routing, events, audit...
}

// After: Framework patterns
export class MyApp extends DO {
  // Built-in Things, Actions, Events, REST API...
}
```
