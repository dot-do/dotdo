# dotdo init

Initialize a new dotdo project with all the scaffolding you need to build your AI-powered startup.

## Usage

```bash
npx dotdo init [project-name] [options]
```

## Arguments

| Argument | Description | Default |
|----------|-------------|---------|
| `project-name` | Name for your project directory | Current directory (`.`) |

## Options

| Option | Description |
|--------|-------------|
| `-t, --template <name>` | Use a specific template |
| `--typescript` | Use TypeScript (default) |
| `--no-git` | Skip git initialization |
| `--no-install` | Skip dependency installation |
| `-y, --yes` | Skip prompts, use defaults |

## Generated Project Structure

After running `init`, you get:

```
my-startup/
  src/
    startup.ts      # Your Startup class
    index.ts        # Worker entry point (Hono HTTP router)
  wrangler.jsonc    # Cloudflare Workers config
  package.json      # Dependencies
  tsconfig.json     # TypeScript config
  .gitignore        # Git ignore rules
  .dev.vars.example # Environment variables template
```

### Key Files

#### src/startup.ts

Your main startup logic:

```typescript title="src/startup.ts"
import { Startup } from 'dotdo'
import { priya, ralph, tom, mark, sally } from 'agents.do'

export class MyStartup extends Startup {
  hypothesis = 'Your hypothesis here'

  async launch() {
    const spec = priya`define the MVP for ${this.hypothesis}`
    let app = ralph`build ${spec}`

    do {
      app = ralph`improve ${app} per ${tom}`
    } while (!await tom.approve(app))

    mark`announce the launch`
    sally`start selling`
  }
}
```

#### src/index.ts

HTTP entry point:

```typescript title="src/index.ts"
import { Hono } from 'hono'
import { MyStartup } from './startup'

const app = new Hono()

app.post('/launch', async (c) => {
  const id = c.env.STARTUP.idFromName('default')
  const stub = c.env.STARTUP.get(id)
  await stub.launch()
  return c.json({ status: 'launched' })
})

export default app
export { MyStartup }
```

#### wrangler.jsonc

Cloudflare configuration:

```json title="wrangler.jsonc"
{
  "name": "my-startup",
  "main": "src/index.ts",
  "compatibility_date": "2024-01-01",
  "durable_objects": {
    "bindings": [
      { "name": "STARTUP", "class_name": "MyStartup" }
    ]
  },
  "migrations": [
    { "tag": "v1", "new_classes": ["MyStartup"] }
  ]
}
```

## Post-Init Setup

### 1. Configure Environment Variables

Copy the example file and add your API keys:

```bash
cp .dev.vars.example .dev.vars
```

Edit `.dev.vars`:

```bash title=".dev.vars"
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
```

<Callout type="error">
Never commit `.dev.vars` to git. It contains secrets.
</Callout>

### 2. Update Your Hypothesis

Open `src/startup.ts` and change the hypothesis:

```typescript
export class MyStartup extends Startup {
  hypothesis = 'Small teams need a simple way to track customer feedback'
  // ...
}
```

### 3. Start Development

```bash
npx dotdo dev
```

Open [http://localhost:8787](http://localhost:8787) to see your startup dashboard.

## Configuration Options

The `init` command creates sensible defaults, but you can customize:

### TypeScript Configuration

```json title="tsconfig.json"
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "types": ["@cloudflare/workers-types"]
  }
}
```

### Package.json Scripts

```json title="package.json"
{
  "scripts": {
    "dev": "dotdo dev",
    "build": "dotdo build",
    "deploy": "dotdo deploy",
    "typecheck": "tsc --noEmit"
  }
}
```

## Troubleshooting

### "Project directory already exists"

The directory must be empty or not exist. Either:

```bash
# Use a different name
npx dotdo init my-other-startup

# Or clear the directory
rm -rf my-startup && npx dotdo init my-startup
```

### "Failed to install dependencies"

Try installing manually:

```bash
cd my-startup
npm install
```

### "Permission denied"

On macOS/Linux, you may need to adjust permissions:

```bash
chmod -R 755 my-startup
```
