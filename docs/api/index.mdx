# API Reference

The dotdo platform exposes a comprehensive REST API built on [Hono](https://hono.dev) running on Cloudflare Workers. All endpoints support JSON request/response bodies and follow RESTful conventions.

## Base URL

```
Production: https://api.dotdo.dev
Development: http://localhost:8787
```

## API Endpoints Overview

| Route | Description |
|-------|-------------|
| `/api/*` | REST API for Things and resources |
| `/api/browsers/*` | Browser automation sessions |
| `/api/sandboxes/*` | Code sandbox execution |
| `/api/obs/*` | Observability and tracing |
| `/mcp/*` | Model Context Protocol (MCP) |
| `/rpc/*` | WebSocket RPC with promise pipelining |
| `/:doClass/:id/*` | Direct Durable Object access |

## Quick Start

### Create a Thing

```bash
curl -X POST https://api.dotdo.dev/api/things \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"name": "My First Thing"}'
```

### List Things

```bash
curl https://api.dotdo.dev/api/things \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Get a Specific Thing

```bash
curl https://api.dotdo.dev/api/things/123e4567-e89b-12d3-a456-426614174000 \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Core Resources

### Things

Things are the fundamental resource type in dotdo. Every entity in the system is represented as a Thing with a consistent schema.

```typescript
interface Thing {
  id: string           // UUID
  $id: string          // Qualified identifier: "thing:{id}"
  $type: string        // Type URL or "thing"
  name: string         // Display name
  data?: object        // Arbitrary JSON data
  createdAt: string    // ISO 8601 timestamp
  updatedAt: string    // ISO 8601 timestamp
}
```

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/things` | List all things (paginated) |
| POST | `/api/things` | Create a new thing |
| GET | `/api/things/:id` | Get a specific thing |
| PUT | `/api/things/:id` | Update a thing |
| DELETE | `/api/things/:id` | Delete a thing |

### Pagination

List endpoints support pagination via query parameters:

```bash
# Get 10 items starting from offset 20
curl "https://api.dotdo.dev/api/things?limit=10&offset=20"
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `limit` | integer | 100 | Maximum items to return (max 1000) |
| `offset` | integer | 0 | Number of items to skip |
