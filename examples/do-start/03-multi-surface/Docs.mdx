## Project Structure

This example includes all five surfaces:

```
03-multi-surface/
  Site.mdx          # Marketing at /
  App.tsx           # Dashboard at /app
  Admin.tsx         # Admin panel at /admin
  Docs.mdx          # Docs at /docs (you are here)
  Blog.mdx          # Blog at /blog
  docs/             # Documentation content
    getting-started.mdx
    surfaces.mdx
    api.mdx
  blog/             # Blog posts
    hello-world.mdx
    multi-surface-architecture.mdx
```

## How It Works

Each surface file is automatically discovered by the CLI:

1. **Site.mdx** takes priority for the root `/` path
2. **App.tsx** is available at `/app`
3. **Admin.tsx** is available at `/admin`
4. **Docs.mdx** is available at `/docs`
5. **Blog.mdx** is available at `/blog`

All surfaces share the same Durable Objects and backend logic.
